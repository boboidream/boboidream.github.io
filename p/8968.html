Caddy + Hexo 实现自动化部署

Aug 26, 2019 | 网站建设 | 阅读

之前博客使用在 VPS 设置 git server 来解决 Hexo 博客发布到 VPS 问题，见文章 使用 VPS 让 Hexo 博客快的飞起，但依然不够方便：

  1. 配置繁琐，需要在 VPS 设置 git 账户，还要做好与本地 SSH 通信
  2. 不够自动，每次发布时需要手动 git push 到 VPS 的仓库

用上 Caddy 后，发现了个令人惊喜的插件 http.git， 可以轻松通过 GItHub webhooks 实现监听 GitHub 仓库 git push 实现自动更新 VPS 内容，甚至还能写其他的部署命令。使用此配置方案，比原方案更加简单，也更省心。以下为配置步骤。

一、准备工作

  1. 首先要有台 VPS 服务器，并且安装好 Caddy Server，并且正常运行。教程参考：用 Caddy 替代 Nginx 迈入 HTTPS 时代
  2. 有 GitHub 仓库托管的 Hexo 博客。教程参考：Hexo + GitHub 搭建个人博客及反思

二、配置 Caddyfile

如果你参考了第一步的教程，那么 Caddy 是自动安装了 http.git 插件的，那么我们只需要更新 Caddyfile。

四、配置 GitHub webhooks

  1. 访问 https://github.com/settings/keys 将 id_rsa_pub 添加到 SHH key。

  2. 进入仓库，进入 Setting/Webhooks ，点击 Add webhook。

  3. Payload URL 填写 网址/webhooks ，Secret 与 Caddyfile 中 password 保持一致。Content type 选 application/json 其他保持默认，点击完成。
